# Концептуальная модель данных

## Назначение концептуальной модели

Концептуальная модель описывает предметную область интернет-магазина на уровне бизнес-сущностей и связей между ними. На этом уровне не важны типы данных, индексы и технические ограничения. Главная задача — показать, какие объекты существуют в бизнесе и как они взаимодействуют.

## Сущности

| Сущность | Описание |
|---|---|
| `Marketing Channel` | Канал привлечения клиента |
| `Customer` | Клиент интернет-магазина |
| `Category` | Категория товара |
| `Product` | Товар из ассортимента |
| `Order` | Заказ клиента |
| `Order Item` | Позиция в составе заказа |
| `Payment` | Оплата заказа |
| `Delivery` | Доставка заказа |
| `Review` | Отзыв клиента о товаре |

## Описание сущностей

### Marketing Channel

Канал, через который клиент пришел в интернет-магазин. Используется для анализа привлечения и сравнения вклада каналов в клиентов, заказы и выручку.

Примеры каналов: `organic_search`, `social_media`, `context_ads`, `email_marketing`, `referral`, `direct`, `marketplace`, `mobile_app`.

### Customer

Клиент интернет-магазина. Хранит базовые атрибуты клиента: имя, фамилию, пол, дату рождения, город, дату регистрации и канал привлечения.

### Category

Группа товаров. Используется для анализа ассортимента на уровне направлений: `Electronics`, `Home Appliances`, `Clothing`, `Shoes`, `Beauty`, `Sports`, `Books`, `Accessories`, `Furniture`, `Kids`.

### Product

Товар, который можно купить в интернет-магазине. У товара есть категория, цена, себестоимость и признак активности.

### Order

Центральная бизнес-сущность модели. Заказ фиксирует факт покупки или попытки покупки клиента и связывает между собой клиента, товары, оплату, доставку и отзывы.

### Order Item

Позиция заказа. Нужна, потому что один заказ может содержать несколько товаров, а один товар может входить во множество заказов.

### Payment

Оплата заказа. Используется для проверки финансового статуса заказа и корректности анализа выручки.

### Delivery

Доставка заказа. Позволяет анализировать выполнение логистики, задержки и влияние доставки на клиентский опыт.

### Review

Отзыв клиента о товаре в рамках заказа. Используется для анализа удовлетворенности клиентов, рейтингов товаров и негативного опыта.

## Концептуальные связи

| Связь | Тип | Описание |
|---|---|---|
| `Marketing Channel` -> `Customer` | 1:N | Один канал может привести много клиентов |
| `Customer` -> `Order` | 1:N | Один клиент может оформить много заказов |
| `Category` -> `Product` | 1:N | В одной категории может быть много товаров |
| `Order` -> `Order Item` | 1:N | Один заказ содержит одну или несколько позиций |
| `Product` -> `Order Item` | 1:N | Один товар может встречаться в разных заказах |
| `Order` -> `Payment` | 1:1 | У заказа есть одна запись об оплате |
| `Order` -> `Delivery` | 1:1 | У заказа есть одна запись о доставке |
| `Customer` -> `Review` | 1:N | Один клиент может оставить несколько отзывов |
| `Product` -> `Review` | 1:N | У одного товара может быть много отзывов |
| `Order` -> `Review` | 1:N | В рамках заказа может быть несколько отзывов |

## Текстовая схема связей

```text
Marketing Channel 1 ── N Customer 1 ── N Order 1 ── N Order Item N ── 1 Product N ── 1 Category
                                      │
                                      ├── 1 Payment
                                      ├── 1 Delivery
                                      └── N Review N ── 1 Product
```

## Почему модель построена вокруг заказа

Заказ — основной факт в e-commerce-аналитике. Именно заказ связывает:

- клиента, который совершает покупку;
- товары, которые были куплены;
- оплату, которая подтверждает финансовый результат;
- доставку, которая влияет на клиентский опыт;
- отзывы, которые помогают оценить качество товара и сервиса.

Поэтому большая часть аналитики строится от `orders` и `order_items`: выручка, прибыль, средний чек, повторные покупки, категории, товары, каналы и доставка.

## Роли ключевых сущностей

- Клиент отвечает на вопрос “кто покупает”.
- Товар отвечает на вопрос “что покупают”.
- Заказ отвечает на вопрос “когда и в каком составе покупают”.
- Оплата отвечает на вопрос “была ли покупка финансово подтверждена”.
- Доставка отвечает на вопрос “насколько качественно выполнен сервис”.
- Отзыв отвечает на вопрос “как клиент оценил опыт после покупки”.
