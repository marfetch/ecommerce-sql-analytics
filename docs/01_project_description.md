# Описание проекта

## Название проекта

**E-commerce SQL Analytics**

## Цель

Создать аналитическую базу данных интернет-магазина в PostgreSQL, наполнить ее синтетическими данными и подготовить SQL-анализ продаж, клиентов, товаров, маркетинговых каналов, доставок и качества данных.

## Бизнес-контекст

Интернет-магазин продает товары из разных категорий и получает клиентов из нескольких каналов привлечения: органический поиск, социальные сети, контекстная реклама, email-маркетинг, реферальный трафик, прямые заходы, маркетплейсы и мобильное приложение.

Для управления бизнесом нужны регулярные ответы на вопросы о выручке, прибыльности, клиентской базе, эффективности каналов, ассортименте и логистике. Проект показывает, как такие вопросы можно решать на уровне SQL без готовой BI-системы.

## Постановка задачи

Необходимо:

1. Спроектировать структуру реляционной базы данных.
2. Создать таблицы, ключи и ограничения.
3. Наполнить справочники и сгенерировать тестовые данные.
4. Создать индексы для типовых аналитических запросов.
5. Подготовить представления для анализа.
6. Реализовать функции и процедуры на PL/pgSQL.
7. Написать аналитические SQL-запросы.
8. Подготовить проверки качества данных.
9. Описать результаты анализа в формате бизнес-отчетов.

## Границы проекта

В проект входят:

- модель данных интернет-магазина;
- синтетические данные;
- SQL-скрипты для создания и наполнения БД;
- аналитические запросы;
- проверки качества данных;
- документация и шаблоны отчетов.

В проект не входят:

- реальные персональные данные;
- полноценная витрина маркетинговых расходов;
- BI-дашборд;
- backend или frontend приложения;
- интеграция с внешними API.

## Основные бизнес-вопросы

- Какова общая выручка, валовая прибыль и средний чек?
- Как меняется выручка по месяцам?
- Какие товары и категории наиболее значимы для бизнеса?
- Какие клиенты покупают повторно?
- Какие сегменты клиентов можно выделить по RFM?
- Какие каналы приводят больше клиентов и выручки?
- Насколько стабильно выполняются доставки?
- Какие заказы связаны с негативным клиентским опытом?
- Какие ошибки или аномалии есть в данных?

## Используемые технологии

- PostgreSQL
- SQL
- PL/pgSQL
- DBeaver
- psql
- Markdown

## Входные данные

Входные данные формируются внутри проекта:

- справочник маркетинговых каналов;
- справочник категорий товаров;
- синтетические клиенты;
- синтетические товары;
- заказы и состав заказов;
- оплаты;
- доставки;
- отзывы.

## Выходные артефакты

- DDL/DML-скрипты в папке `sql`;
- аналитические представления;
- SQL-функции и хранимые процедуры;
- набор аналитических запросов;
- проверки качества данных;
- документация модели данных в папке `docs`;
- аналитические отчеты в папке `results`.

## Ожидаемый результат

После запуска SQL-скриптов пользователь получает базу данных, на которой можно выполнять аналитические запросы и заполнять отчеты фактическими значениями. Проект демонстрирует полный цикл работы аналитика с реляционной базой: от модели данных до бизнес-интерпретации.
